üìä Understanding Tables & Relations

Table Terminology
In DBMS, data is strictly organized in row and column format with specific terminology
Roll Name Age
1 ABC 19
2 DEF 22
3 XYZ 28
üìã Columns (Vertical)
Also called: Fields, Attributes, or Domains
All data in one column must be the same type
Example: "Age" column contains only integers
üìù Rows (Horizontal)
Also called: Tuples or Records
Data items in one row can be different types
Example: One row has int, string, int (1, "ABC", 19)
üí° Key Insight
Columns are homogeneous (same data type within a column)
Rows are heterogeneous (different data types across columns)

üéØ Why This Matters

  • Organized Storage: Data is structured, not random files
  • Relationships: Can connect related information across tables
  • Efficiency: Quick retrieval through proper organization
  • Multi-User: Different people can access simultaneously
  • Security: Control who sees what data
  • Integrity: Maintain data accuracy and consistency
